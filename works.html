01 / We Bare Bears the Puzzle
월드맵 기획 · 미션 연출 · 시나리오

위베어 베어스 IP 기반 퍼즐 게임 월드맵 스토리 구조 총괄. 게임플레이 동선과 서사를 하나의 흐름으로 연결하는 구조 설계 담당.

플랫폼 모바일 (iOS / Android)
엔진 Unity3D
주요 역할 시나리오, 미션 연출, 데이터 세팅
스크립팅 Lua
심층 케이스

놈놈 저택 · 가방 되찾기 미션

디자인 의도

악당 추적 스토리에서 자연스럽게 맵을 순회하도록 미션을 배치. 새로운 장소로 이동하기 전에 유저들이 직접 꾸민 공간을 한 번 더 둘러보게 하여, 이야기 마무리와 공간적 완결감을 동시에 제공한다.

한 파트 미션 전체 단독 설계 사례. 시나리오 집필 및 미션 연출 전담. 플레이어 동선, 감정 곡선, IP 세계관 세 가지 제약을 동시에 만족하는 구조 설계.

축소 결정

원래는 탐문 형식의 미션을 더 추가해 추리 느낌을 살리려 했으나, 재화로 진행하는 게임 특성상 억지로 끌어가는 느낌을 줄 수 있어 단일 미션으로 축소. 대신 각 장면에 밀도를 높이는 방향으로 전환.


아이스베어 탐문 장면: 대화 능력이 부족한 아이스베어가 탐문을 맡는 상황을 설정. 대화가 어긋나는 것처럼 보이지만 결과적으로 원하는 정보에 도달하는 방식으로 개그를 유도.


아이들 오해 장치: 이전 스토리에서 어른들이 아이들의 놀이터를 철거하려 한 맥락을 활용. 악당을 철거하러 온 어른으로 오해한 아이들이 가방도 뇌물로 착각하게 만들어 자연스러운 협력 구도 형성.


캐릭터 역할 분배: 판다가 재치로 악당을 검거하는 장면 구성을 위해 아이스베어는 기습 무력화, 그리즐리는 추격 당하는 상태로 설정하여 서사적 공간 확보.

그리즐리 악당에게 쫓기는 상태
아이스베어 기습으로 무력화
판다 재치로 악당 검거

쿼터뷰 연출 적응: 원작의 기습 장면은 쿼터뷰에서 직접 표현이 어려워, 순간이동 연출로 변형하여 적용.

원작에서는 아이들이 범인을 붙잡으나, 게임에서는 주인공이 직접 잡는 것이 클라이맥스에 맞다고 생각하여 각색. 하지만 곰들이 직접 사람을 공격하는 것은 원작의 성격과 맞지 않아, 테니스공 발사기라는 도구를 활용한 간접 공격 연출로 대체.

02 / Unbound
Unbound · 1인 개발 프로토타입 · 2024

UE5 Blueprint를 활용한 쿼터뷰 1인 개발 프로토타입. 캐릭터 이동, NPC 인터랙션, 대화 시퀀스, 퀘스트 수락 등 게임 로직 전반을 Blueprint로 구현.

레벨 에셋은 Unreal Marketplace에서 조달. 캐릭터 이미지는 GPT IMAGE 생성 후 Photoshop 가공, 사운드는 Audacity 편집.

제작년도 2024
형태 프로토타입
엔진 / 스크립트 Unreal Engine 5 / Blueprint
도구 Blender, Photoshop CC, Audacity, GPT IMAGE
AI 설계

보스 AI

보스 캐릭터의 AI를 Behavior Tree + Blackboard로 설계. 상태 기반(Passive / Investigating / Attacking) 분기 구조로, HP 페이즈에 따라 공격 패턴이 전환된다.

Behavior Tree

루트 Selector에서 AI 상태별로 분기한다. Passive 상태에서는 순찰/배회, Investigating 상태에서는 마지막 감지 위치로 이동, Attacking 상태에서는 거리와 HP에 따라 공격 패턴을 선택한다.

공격 패턴사거리설명
Swing / Slam근접기본 근접 공격 콤보
Charge중거리돌진 후 텔레포트 연계
Orb Shoot원거리마법 투사체 2연발
Blackhole원거리광역 흡인 + 데미지
Summon자체HP 페이즈 시 미니언 6체 소환
Blackboard

Blackboard에 6개 키를 정의하여 AI 판단에 사용한다. AIPerception으로 플레이어를 감지하면 AttackTarget과 State를 갱신하고, Decorator가 이 값을 읽어 행동을 분기한다.

키타입용도
AttackTargetObject현재 공격 대상 (플레이어)
StateEnumAI 상태 (Passive / Attacking / Investigating)
PointOfInterestVector조사할 관심 지점
LastKnownLocationVector타겟 마지막 위치
공격 패턴 시연

Behavior Tree에서 정의된 각 공격 패턴의 인게임 동작.

일반 전투
격노 페이즈 (HP 임계치 이하)

Summon

HP 임계치 이하로 진입하면 텔레포트 후 Skeleton Knight + Archer 미니언을 최대 6체 소환한다.

격노 콤보 (BTD_Boss_HP 트리거): Teleport → Summon x6 → Orb Shoot → Blackhole 를 연속 실행.

소환 직후 본체는 속도가 상승하며, 미니언이 플레이어를 포위하는 동안 원거리 공격으로 압박한다.

구현 상세

플레이어 Blueprint (ET_TopDown_Player)

Enhanced Input으로 이동·공격·구르기·상호작용 4가지 입력을 처리하고, 3단계 콤보 공격·구르기·피격 반응 등 전투 로직을 Blueprint로 구현. NPC 대화, 게임오버 UI 연동까지 플레이어 관련 로직 전반을 담당한다.

입력 & 전투

03 / FarmQuest
퀘스트 시스템 습작 · 1인 개발 · 2026

생성형 AI 툴을 활용한 게임 개발 파이프라인 실험 및 데이터 기반 퀘스트 시스템 설계 습작. 이미지, 사운드, 3D 모델, 코드 작업 전반에 AI 툴 적용.

제작년도 2026
형태 프로토타입
사용 툴

Unity 6 · GPT IMAGE · Adobe Photoshop · MeshyAI · Elevenlabs · Audacity · CursorAI

소개 문서
데이터 구성

퀘스트 데이터

퀘스트 데이터를 JSON으로 구조화하여 기획과 구현 분리. 퀘스트 ID, NPC 참조, 시작 조건, 목표 단계, 보상까지 모든 정보를 데이터로 관리.

필드타입설명
idstring퀘스트 고유 ID
titleIdstring로케일 시스템 참조 제목 ID
providingNPCIdstring퀘스트를 제공하는 NPC ID
completionNPCIdstring퀘스트 완료를 처리하는 NPC ID
startConditionsarray퀘스트 시작 조건 목록
objectivesarray퀘스트의 목표 단계 리스트
rewardsarray퀘스트 완료 보상 목록

startConditions로 선행 퀘스트 완료, 플래그 값 검사, 아이템 보유 여부 등 복수 조건 설정 가능. objectives는 순차 진행되며, 지역 도달(EnterArea), 대화 완료(CompleteDialogue), 적 처치(DefeatEnemies), 아이템 수집(CollectItems) 네 가지 타입 지원.

{
  "id": "quest_potatostew_01",
  "titleId": "Quest_title_potatostew",
  "providingNPCId": "npc_roy",
  "completionNPCId": "npc_roy",
  "startConditions": [],
  "startDialogueId": "conv_potatostew_01",
  "completionDialogueId": "conv_potatostew_05",
  "objectives": [ ... ],
  "rewards": [
    { "itemId": "Item_0001", "amount": 100 },
    { "itemId": "Item_0002", "amount": 100 }
  ]
}
데이터 구성

대화 데이터

캐릭터 간 텍스트 대화를 데이터로 제어. 캐릭터 이름, 초상화(감정), 음성, 애니메이션 연출을 각 대사 라인별로 설정 가능하며, 대화 종료 시 플래그 변경을 통해 스토리 및 퀘스트 진행 제어.

필드타입설명
textIdstring텍스트 대사 로케일 ID
characterIdstring대사를 하는 캐릭터 ID
animationstring캐릭터 애니메이션
expressionstring캐릭터 초상화 ID
voicestring캐릭터 음성 ID
flagActionsarray대화 종료 후 변경할 플래그

flagActions는 네 가지 액션 타입 지원: 플래그 설정(0), 값 증가(1), 플래그 토글(2), 플래그 제거(3). 대화만으로도 퀘스트 상태 전환, NPC 행동 변화 등 유연하게 제어 가능.

{
  "id": "conv_roy_default",
  "lines": [
    {
      "characterId": "npc_roy",
      "expression": "idle",
      "textId": "dlg_roy_default_dlg_01",
      "voice": "talk",
      "animation": "Talk_1",
      "flagActions": [
        {
          "type": 0,
          "key": "potatostew_phase",
          "value": 7
        }
      ]
    }
  ]
}
데이터 구성

캐릭터 데이터

NPC의 표정, 음성, 기본 대화, 플래그 조건부 대화를 데이터로 관리. dialogueConditions를 통해 플래그 값에 따라 NPC 재생 대화를 자동 전환, 별도 코드 없이 NPC 상태 변화 표현.

필드타입설명
characterIdstring캐릭터 고유 ID
nameKeystring이름의 로케일 키
defaultDialogueIdstring조건 없을 때 기본 재생 대화 ID
greetingLocaleIdstringNPC에게 말 걸었을 때 인삿말 로케일
expressionsarray감정 목록 (key + address)
voicesarray음성 목록 (key + address)
dialogueConditionsarray플래그 값에 따라 재생될 대화 조건
 
{
  "characterId": "npc_roy",
  "nameKey": "npc_01_name",
  "defaultDialogueId": "conv_roy_default",
  "greetingLocaleId": "NPC_01_interact_01",
  "expressions": [
    { "key": "idle",     "address": "Roy_idle" },
    { "key": "thinking", "address": "Roy_think" }
  ],
  "voices": [
    { "key": "talk", "address": "Roy_talk" }
  ],
  "dialogueConditions": [
    {
      "flagId": "potatostew_phase",
      "state": 1,
      "dialogueId": "conv_potatostew_10"
    }
  ]
}
